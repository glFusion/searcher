glFusion Searcher Plugin Configuration

The Searcher Plugin provides an improved search engind for glFusion that indexes documents and returns results weighted by relevance.

Main Searcher Options

Display Name
Enter the display name to be shown on administration and other pages.
Minimum Word Length to Consider
Enter the minimum number of letters to be considered a "word". Words with less than this number of letters will not be added to the index.
Max words in a phrase
Enter the maximum number of words to be indexed and searched as a phrase. Increasing the value of this number will significantly increase your index size and indexing time required, but will provide better search results based on phrases entered. You must re-index all content for a new value to take effect.
Results to show per page
Enter the number of search results to show on each page.
Length of excerpt to dispaly
Enter the number of words to display in the excerpt on the search results page.
Maximum occurrences of a term to count
Enter the maximum number of times that a single word or phrase will be counted in a page when calculating the weights. This is to keep some pages which may have many occurrences of a single word from always appearing at the top of the search results, outweighing other pages that may be more relevant.
Show author name in results
Select whether the Author Name should be shown in the results and, if so, whether to link to the author's profile.
Stemmer
(Experimental) Select whether to use a language stemmer. The indexes must be recreated if this option is changed.
Ignore Autotags?
Select "Yes" to ignore the raw content of autotags when indexing and searching. If this is set to "No" then the text in autotags is indexed and searched. Note that autotags are not processed in any case.
Replace Stock glFusion Search?
Select "Yes" for the Searcher plugin to replace the standard glFusion search function. If set, the Search plugin will be used anyplace "search.php" is called, such as the search form in the header and links from profile pages. If this is set to "No" then the standard search function is used. The Searcher can still be accessed at /searcher/index.php.
Queue Indexing?
Select "Yes" to use the LGLib job queue to batch-process indexing jobs. This will make the site more responsive when saving documents. Select "No" to have documents indexed immediately as they are saved.

Searcher Weightings

Weighting for Content
Select the weight (relevance) that will be applied for search terms found in the document content. A higher value causes terms in this area to be considered more "relevant" than those found in other fields.
Weighting for Author
Select the weight (relevance) that will be applied for search terms found in the document's Author field. A higher value causes terms in this area to be considered more "relevant" than those found in other fields.
Weighting for Title
Select the weight (relevance) that will be applied for search terms found in the document's Title field. A higher value causes terms in this area to be considered more "relevant" than those found in other fields.
